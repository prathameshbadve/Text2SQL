Information on all tables in the database -

1. Customers
    - name: Customers
    - summary: Contains information about all customers; name, email, phone
    - purpose: Single source of truth about the customers' basic information.
    - columns: (customer_id, name, email, phone)
    - dependencies_thoughts: customer_id field will be used to join to other tables.
    - keys: customer_id is the primary key for this table and will serve the foreign key in multiple other tables.
    - connected_tables: Sales table connected via customer_id
    - entities: Customer contact details can be extracted from this table.
    - strong_entities: 
    - DDL:
        """
            CREATE TABLE IF NOT EXISTS Customers (
                customer_id INTEGER PRIMARY KEY AUTOINCREMENT,
                name TEXT NOT NULL,
                email TEXT UNIQUE NOT NULL,
                phone TEXT
            );
        """

2. Sales
    - name: Sales
    - summary: Contains details on all sales done in the business.
    - purpose: Single source of truth on all sales transactions done.
    - columns: (sale_id, customer_id, product, quantity, sale_date)
    - dependencies_thoughts: customer_id field is used to join with the Customers table.
    - keys: sale_id is the primary key for this table, customer_id is the foreign key
    - connected_tables: Customers table connected via customer_id
    - entities: Quantity of products sold, name of products sold, day of sales, etc.
    - strong_entities: Total quanity of products sold.
    - DDL:
        """
            CREATE TABLE IF NOT EXISTS Sales (
                sale_id INTEGER PRIMARY KEY AUTOINCREMENT,
                customer_id INTEGER NOT NULL,
                product TEXT NOT NULL,
                quantity INTEGER NOT NULL,
                sale_date TEXT NOT NULL,
                FOREIGN KEY (customer_id) REFERENCES Customers(customer_id)
            );
        """
